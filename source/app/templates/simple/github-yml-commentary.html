The --raise-cop-error flag on line 98 means the rubocop call will return a non-zero exit code if any lint violations are found. This means:
  • when there are no rubocop lint violations, the kosli attest command on line 100 will run, with the default --compliant=true
  • when there are rubocop lint violations, the kosli attest command will not run. The dashboard.lint evidence will be missing, causing the overall compliance of the dashboard artifact to be incompliant
To explicitly attest incompliant evidence, you can run rubocop like this:
set +e rubocop --raise-cop-error . > "${LINT_EVIDENCE_DIR}"/rubocop.log STATUS=$? set -e
set the KOSLI_LINT_COMPLIANT flag based on $STATUS:
KOSLI_LINT_COMPLIANT=$([ ${STATUS} = 0 ] && echo true || echo false)
and run kosli attest with the --assert flag:
kosli attest generic \ --assert \ --compliant="${KOSLI_LINT_COMPLIANT}" \ --evidence-paths="${LINT_EVIDENCE_DIR}" \ --name=dashboard.lint
