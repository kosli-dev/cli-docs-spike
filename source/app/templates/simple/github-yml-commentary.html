kosli attest generic

Line 101 is an example of a call to kosli attest generic. It attests the outcome of the Rubocop Ruby style linter. The Rubocop --raise-cop-error flag on line 99 is important:
  • when there are no Rubocop lint violations:
    • the exit-code will be zero.
    • the kosli attest generic command will run.
    • its --compliant flag will default to true.
    • the dashboard.lint evidence will be true.
  • when there are Rubocop lint violations:
    • the exit-code will be non-zero.
    • the kosli attest generic command will not run.
    • the dashboard.lint evidence will be missing.
Since only a compliant dashboard artifact will get deployed (thanks to the Kosli sdlc-control-gate on line 223), and your auditors will only need proof for deployed artifacts, this is perfectly acceptable. However, if you wish to explicitly attest incompliant evidence, you can run Rubocop like this:
set +e rubocop --raise-cop-error . | tee ${KOSLI_EVIDENCE_PATHS}/rubocop.log STATUS=${PIPESTATUS[0]} set -e
set the KOSLI_COMPLIANT flag based on ${STATUS} like this:
KOSLI_COMPLIANT=$([ ${STATUS} = 0 ] && echo true || echo false)
and run kosli attest generic as before:
kosli attest generic \ --evidence-paths=${KOSLI_EVIDENCE_PATHS} \ --name=dashboard.lint
If you want the lint: job to return Rubocop's exit status, add a final exit ${STATUS}.
